# Obsidian Markdown Concatenation Utility

Консольная утилита для объединения всех Markdown-файлов из Obsidian vault в один файл.

## Описание

Утилита рекурсивно сканирует указанную папку, находит все файлы с расширением `.md` и объединяет их содержимое в один выходной файл. Каждый файл отделяется заголовком с его относительным путём.

## Установка

Утилита использует только стандартную библиотеку Python. Требуется Python 3.6 или выше.

```bash
# Клонировать репозиторий
git clone <repository-url>
cd obsidian-concat

# Проверить версию Python
python --version
```

## Использование

### Базовое использование

```bash
# Объединить все .md файлы из текущей директории в combined.md
python concat.py

# Указать входную папку
python concat.py --input sample

# Указать входную папку и выходной файл
python concat.py --input sample --output result.md

# Короткая форма аргументов
python concat.py -i ./notes -o combined_notes.md
```

### Аргументы командной строки

- `-i, --input` — Путь к папке для сканирования (по умолчанию: текущая директория)
- `-o, --output` — Путь к выходному файлу (по умолчанию: `combined.md`)
- `-h, --help` — Показать справку

### Примеры

```bash
# Обработать папку sample и сохранить результат в текущей директории
python concat.py --input sample

# Обработать папку notes и сохранить в specific_folder/output.md
python concat.py -i notes -o specific_folder/output.md

# Обработать текущую папку
python concat.py
```

## Функциональность

### Основные возможности

- ✅ Рекурсивное сканирование всех подпапок
- ✅ Обработка всех файлов с расширением `.md`
- ✅ Поддержка UTF-8 и кириллицы
- ✅ Разделители между файлами с указанием пути
- ✅ Обработка ошибок доступа и кодировки

### Обработка ошибок

Утилита обрабатывает следующие ситуации:
- Несуществующая входная директория
- Отсутствие прав на чтение/запись
- Отсутствие `.md` файлов в указанной папке
- Ошибки кодировки (файлы с невалидным UTF-8)
- Попытка включения выходного файла в конкатенацию (автоматически исключается)

## Структура выходного файла

Каждый файл в результирующем `combined.md` разделяется заголовком:

```markdown
================================================================================
File: AWS Access Context.md
================================================================================

[содержимое файла]


================================================================================
File: Iteration1/Deployment — Inbound Lambda.md
================================================================================

[содержимое файла]
```

## Тестирование

Протестировать утилиту на примере данных:

```bash
python concat.py --input sample --output test_output.md
```

Папка `sample/` содержит примеры Markdown-файлов из Obsidian vault для проверки работы утилиты.
